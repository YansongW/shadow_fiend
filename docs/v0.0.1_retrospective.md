## 1. 版本目标

验证 shadow_fiend 在 macOS Apple Silicon 上的端到端可行性：

> 本地播放器声音 → BlackHole 2ch → 音频捕获 → VAD 切句 → SenseVoice ASR → Argos 翻译 → PyQt6 字幕浮窗

| 模块 | 状态 | 说明 |
|------|------|------|
| 音频捕获 | ✅ | PyAudio + BlackHole 2ch，支持设备原生采样率采集与软件重采样 |

## 3. 存在的问题

### 3.1 延迟偏高（严重影响观影体验）

实测各环节耗时：

| 环节 | 耗时 | 影响 |
|------|------|------|
| 首次启动加载 SenseVoice 模型 | 10–13 秒 | 启动延迟，加载完成后不影响实时性 |
| 10 秒语音片段 ASR 推理 | 0.5–2 秒 | 中等，MPS 首次编译慢，后续较快 |
| Argos 翻译（ko/ja → en → zh pivot） | 1–3 秒 | 较大，双段翻译增加延迟与语义损失 |
| **端到端可见字幕延迟** | **约 2–5 秒** | **明显，看快节奏内容会跟不上** |

**结论**：2–5 秒延迟对普通对话剧尚可接受，但对综艺、动作片、快节奏影视剧影响较大。MVE 2 阶段必须把延迟降到 1 秒以内。

### 3.2 翻译质量不稳定

- Argos 缺乏直接的中日韩互译包，强制走 English pivot：
  - 日语例句被识别为长串重复后，翻译结果语义混乱
  - 韩语例句翻译为「这是你好，这是这个。」，丢失「测试」原意
- 口语化、省略主语的日韩语在 pivot 后容易失真

### 3.3 UI/UX 简陋

- 字幕窗口样式不可调（字体、大小、颜色、位置）
- 不支持点击穿透，可能遮挡视频播放控件
- 不支持位置记忆，每次启动回到屏幕底部中央
- 没有字幕历史或导出功能
- 无托盘图标 / 菜单栏入口

### 3.4 进程与资源管理

- 测试过程中多次出现 Qt/PyAudio 进程未正确终止的情况
- 需要 `kill -9` 才能彻底清理
- Python `multiprocessing.resource_tracker` 偶尔残留

### 3.5 环境依赖较重

- 需要用户手动配置 BlackHole 2ch 多输出设备
- SenseVoice 模型首次下载约 1GB，对网络环境有要求
- Python 3.10+、Homebrew、portaudio 等前置依赖对非技术用户不够友好

### 3.6 Logo 仍未定稿

- 当前 SVG 为占位风格化设计，用户对现有方案不完全满意
- 需要继续迭代品牌视觉

## 4. 改进建议

### 4.1 降低延迟（优先级：高）

1. **换用直接中日韩互译模型**
   - 调研 opus-mt、NLLB-200-small、Helsinki-NLP 等支持直接 ja↔zh、ko↔zh 的离线模型
   - 替换 Argos 的 English pivot 路径
2. **流式 ASR 或更短 VAD 窗口**
   - 将 VAD `max_utterance_ms` 从 10 秒缩短到 3–5 秒
   - 探索 SenseVoice 的流式推理或 chunk 级推理能力
3. **模型预热与常驻**
   - 首次启动后可将模型常驻内存，避免重复加载
   - 提供后台服务模式，观影前完成预热
4. **默认 ASR 设备策略**
   - MPS 首次编译慢，可考虑默认 CPU 并在后台异步切换至 MPS

### 4.2 提升翻译质量（优先级：高）

1. 直接中日韩互译模型替代 English pivot
2. 加入上下文感知：保留前 1–2 句作为翻译 prompt
3. 术语表 / 自定义翻译规则（如角色名、专有名词固定译法）

### 4.3 UI/UX 优化（优先级：中）

1. 字幕样式设置面板（字体、大小、颜色、描边、背景透明度）
2. 窗口位置记忆与拖拽吸附
3. 点击穿透模式（鼠标可穿透浮窗操作播放器）
4. 菜单栏小图标 / 全局快捷键显示/隐藏
5. 字幕历史面板与 SRT 导出

### 4.4 稳定性（优先级：中）

1. 统一进程退出逻辑，确保 `pipeline.stop()` 能彻底释放 PyAudio 与 Qt 资源
2. 测试脚本中加入进程泄漏检测
3. 提供 `--headless` 或 `--no-ui` 模式用于后台调试

### 4.5 安装体验（优先级：中）

1. 打包为 `.app` 或 `.dmg`，降低非技术用户门槛
2. 自动检测并提示 BlackHole 配置
3. 模型下载进度可视化

### 4.6 品牌（优先级：低）

1. Logo v2 迭代
2. 项目官网 / Release Notes 美化

## 6. 结论

v0.0.1 成功证明了 shadow_fiend 的技术可行性：本地音频捕获 → ASR → 翻译 → 字幕浮窗的全链路已跑通。但当前版本更像一个「技术验证版」，距离舒适观影还有明显距离。**延迟和翻译质量是最大短板**，应在 v0.0.2 优先解决。
