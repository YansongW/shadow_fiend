# 高质量平行语料获取与清洗方法 v0.0.5

## 目标

为 shadow_fiend 训练低延迟、高质量的 ja→zh / ko→zh / en→zh 翻译模型，构造领域相关、对齐准确、噪音少的平行语料。

## 当前问题

CCMatrix 数据量大但质量参差不齐：
- 网页模板、下划线占位符
- 宗教文本占比高
- 语言识别错误（如日语源句被识别为其他语言）
- 低质量对齐（源句和目标句语义不匹配）

## 数据获取渠道

### 1. 网页挖掘平行语料（已有）

| 数据集 | 语言对 | 规模 | 优点 | 缺点 |
|--------|--------|------|------|------|
| CCMatrix | ja-zh / ko-zh / en-zh | 数百万–数千万 | 量大、易获取 | 噪音大、领域杂 |
| WikiMatrix | 多语言 | 中等 | 质量较高 | 百科文本，口语少 |
| CC-Aligned | 多语言 | 大 | 网页覆盖广 | 质量不稳定 |

### 2. 字幕平行语料（理想领域）

| 数据集 | 语言对 | 规模 | 获取难度 |
|--------|--------|------|----------|
| OpenSubtitles | ja/zh, ko/zh 较少 | 中等 | 需要下载原始字幕并对齐 |
| JESC | ja-en | 中等 | 仅日英，需再转 |
| TED Talks | en-zh 有官方 | 小 | 质量高但量小 |
| 动漫/影视字幕站点 | ja-zh / ko-zh | 不确定 | 版权风险、需爬虫 |

### 3. 人工/半人工数据集

- WMT/CWMT 比赛数据：en-zh 有公开高质量数据
- Tianchi 比赛数据：可能有中日/中韩
- 自行标注：成本最高但质量最好

## 数据清洗方法

### 1. 规则清洗（已实现）

- 空值、长度异常过滤
- 源/目标长度比过滤
- 下划线、特殊符号、数字占比过滤
- 繁体转简体
- 去重
- 宗教/敏感词黑名单

### 2. 语言识别过滤（已实现）

- 使用 fasttext `lid.176` 模型
- 确保源句为 ja/ko/en，目标句为 zh
- 置信度阈值建议 0.8–0.9

### 3. 语义相似度过滤（实现中）

- **LaBSE**：跨语言句子嵌入，计算源/目标余弦相似度
  - 优点：对 ja/zh/ko/en 支持好
  - 缺点：模型较大（~1.8GB），推理较慢
- **LASER**：Meta 出品，专为平行语料设计
  - 优点：轻量、专为对齐设计
  - 缺点：安装复杂、需 fairseq
- **NLLB 对齐分数**：用翻译模型计算 p(y|x)
  - 优点：与最终任务一致
  - 缺点：计算量大

### 4. 领域过滤（待实现）

- 识别对话/字幕风格文本
- 过滤百科、法律、宗教等无关领域
- 可使用轻量级文本分类器或关键词过滤

### 5. 毒性/脏话过滤

- 过滤明显不当内容
- 避免模型学到不良表达

## 推荐清洗 pipeline

```
原始 CCMatrix
  → 规则清洗（长度、符号、下划线、去重）
  → 繁体转简体
  → 语言识别过滤
  → LaBSE 语义相似度过滤
  → 领域/毒性过滤（可选）
  → 高质量训练数据
```

## 质量评估方式

1. **保留率**：清洗后剩余句对比例
2. **人工抽查**：随机抽取 100 条，判断可用比例
3. **LaBSE 分数分布**：高分段比例
4. **下游任务 BLEU**：用清洗后的数据 fine-tune 后评测
