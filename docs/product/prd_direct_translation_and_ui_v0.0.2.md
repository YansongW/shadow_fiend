# shadow_fiend v0.0.2 PRD：流式低延迟翻译 + 托盘式 UI 升级

**版本**：v0.0.2  
**状态**：草案，待用户确认  
**目标用户**：在 macOS 上看日韩影视剧、需要实时翻译字幕的中文用户  
**产品经理**：AI PM  
**日期**：2026-06-25

---

## 1. 背景与目标

### 1.1 背景

v0.0.1 已验证 shadow_fiend 端到端技术链路可行：

- 音频捕获 → VAD → SenseVoice ASR → Argos 翻译 → PyQt6 字幕浮窗

但 v0.0.1 复盘报告指出核心问题：

1. **延迟 2–5 秒**：当前「整句式 batch pipeline」必须等一句话说完才送 ASR，且 Argos 强制走 `ja/ko → en → zh` pivot。
2. **翻译质量不稳定**：日韩语经英语中转后语义丢失、重复句被错误拼接。
3. **UI 体验粗糙**：设置入口在浮窗右键菜单中，打断观影。

### 1.2 目标

v0.0.2 的核心目标是：

> **让 shadow_fiend 从「能跑通」变成「能看剧」：采用流式 pipeline 降低延迟，并提供一个不打扰、美观、易用的字幕体验。**

具体指标：

| 指标 | v0.0.2 目标 | v0.0.3 目标 |
|------|-------------|-------------|
| 端到端字幕延迟 | **300–500ms** | **200–300ms** |
| ASR 架构 | 流式 ASR（Paraformer streaming 或 SenseVoice 短窗口流式） | 稳定流式 + 上下文缓存 |
| VAD | Silero VAD（32ms 级） | 自适应 Silero VAD |
| 翻译 | opus-mt 直接中日韩互译 | 增量翻译 + 上下文感知 |
| UI | 系统托盘/菜单栏图标 + 浮窗精简交互 | 持久化配置 + 主题 |

**项目准线**：严格本地隐私优先，不考虑任何商业云 API。

## 2. 用户故事

### 2.1 核心观影场景

> 小王打开 IINA 播放韩剧，点击菜单栏的 shadow_fiend 小图标启动。剧集开始后，屏幕下方出现半透明双语字幕，几乎与演员口型同步。小王点击托盘图标调整字号和位置，设置面板从托盘展开；他开启「点击穿透」后继续观影，字幕不再遮挡进度条操作。

### 2.2 导出与回看

> 小王想把字幕保存下来，点击托盘图标选择「导出 SRT」，选择保存路径后得到时间轴对齐的字幕文件。

### 2.3 流式字幕体验

> 演员说话时，字幕逐词/逐短语出现；演员停顿后，字幕在 300ms 内完成稳定并显示最终翻译。

## 3. 功能需求

### 3.1 流式低延迟 Pipeline（P0）

#### 3.1.1 架构改造

当前 batch pipeline：

```
音频 → 等一句话说完（VAD） → 整句 ASR → 整句翻译 → 显示
```

v0.0.2 流式 pipeline：

```
音频（32–40ms chunk） → Silero VAD → 流式 ASR → 片段翻译 → 实时滚动字幕
```

#### 3.1.2 ASR 引擎选型

**候选方案**：

| 方案 | 模型 | 流式支持 | 中日韩准确率 | 本地 | 延迟 | 推荐度 |
|------|------|----------|--------------|------|------|--------|
| **Paraformer streaming** | `iic/paraformer-zh` / 多语言流式版 | ✅ 原生 | 中文极强，日韩需确认 | ✅ | **<100ms** | ⭐⭐⭐⭐⭐ |
| **SenseVoice 短窗口** | `iic/SenseVoiceSmall` | ⚠️ 需 chunk 模拟 | 中日韩优秀 | ✅ | 100–300ms | ⭐⭐⭐⭐ |
| **faster-whisper** | `tiny/base` | ⚠️ 需 streaming wrapper | 一般 | ✅ | 100–300ms | ⭐⭐⭐ |

**PM 建议**：

1. 先由 DEV/QC 建立标准测试集（日韩影视片段 + 清晰人声），对比 Paraformer streaming 与 SenseVoice 短窗口方案的 **WER/CER、延迟、资源占用**。
2. 如果 Paraformer streaming 在中日韩场景准确率不低于 SenseVoice 的 95%，则采用 Paraformer。
3. 否则采用 SenseVoice 短窗口流式模拟方案。

#### 3.1.3 VAD 升级

- 能量阈值 VAD 替换为 **Silero VAD**
- 音频 chunk 从 100ms 缩短到 **32–40ms**
- 支持自适应阈值（根据环境噪声动态调整）

#### 3.1.4 翻译引擎

- 使用 **Helsinki-NLP/opus-mt** 直接语言对模型：`ja-zh`、`ko-zh`、`en-zh`
- 片段级翻译：每收到一个语义完整短语即翻译
- 保留 Argos 作为不支持语言对的回退

### 3.2 系统托盘/菜单栏 UI（P0）

#### 3.2.1 设计原则

- **观影时不打扰**：主字幕浮窗无按钮、无菜单栏、无边框
- **设置入口收敛到托盘/菜单栏**：参考网易云音乐、Spotify 等播放器托盘操作
- **小 Logo 常驻**：启动后在菜单栏显示 shadow_fiend 图标

#### 3.2.2 托盘菜单

```
点击菜单栏图标
├── 🎙 开始 / 暂停监听
├── ─────────────
├── 字幕样式...          → 打开样式设置面板
├── 位置：上 / 中 / 下 / 自定义
├── 点击穿透：开 / 关
├── ─────────────
├── 导出 SRT...
├── ─────────────
├── 关于
└── 退出
```

#### 3.2.3 字幕浮窗交互

- 左键拖拽移动位置
- 右键仅保留「样式设置...」和「退出」（可选，主要入口在托盘）
- 点击穿透模式下，浮窗不拦截鼠标，但保留托盘控制

#### 3.2.4 样式设置面板

- 从托盘菜单打开，而非浮窗右键
- 支持：原/译文字号、颜色、背景透明度、预设位置
- 修改实时预览

### 3.3 字幕历史与 SRT 导出（P1）

- Pipeline 维护 `subtitle_history`
- 托盘菜单「导出 SRT...」
- 格式：`HH:MM:SS,mmm`

### 3.4 配置持久化（P1）

- 保存路径：`~/.config/shadow_fiend/settings.json`
- 保存内容：样式、位置、点击穿透状态、源/目标语言
- 下次启动自动恢复

## 5. 技术方案

### 5.1 流式 Pipeline 模块

```python
class StreamingPipeline:
    def __init__(self):
        self.audio_capture = AudioCaptureModule(chunk_duration_ms=32)
        self.vad = SileroVADModule()
        self.asr = StreamingASRModule()  # Paraformer or SenseVoice
        self.translator = OpusEngine()
        self.ui = SubtitleWindow()
        self.tray = SystemTrayController()
```

### 5.2 ASR 接口抽象

```python
class StreamingASRModule(ABC):
    @abstractmethod
    def reset(self): ...

    @abstractmethod
    def feed_audio(self, chunk: np.ndarray) -> Optional[ASRPartialResult]: ...

@dataclass
class ASRPartialResult:
    text: str           # 当前识别文本
    is_final: bool      # 是否稳定
    language: str       # 检测语言
```

### 5.3 增量翻译策略

- 收到 `is_final=True` 的片段立即翻译
- 收到 `is_final=False` 的片段，每 500ms 或遇到标点时翻译一次
- 使用滑动窗口避免重复翻译

## 8. 风险与依赖

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| Paraformer streaming 日韩准确率不达标 | 必须回退到 SenseVoice | 先测试再决策 |
| Silero VAD 对影视背景音敏感 | 误触发或漏检 | 增加噪声门限和自适应阈值 |
| 流式翻译片段不连贯 | 字幕闪烁/语义破碎 | 增量翻译 + 平滑策略 |
| 系统托盘图标在 macOS 上显示异常 | UI 入口缺失 | 使用 PyQt6 QSystemTrayIcon 充分测试 |
| 配置持久化路径权限问题 | 设置无法保存 | 使用标准 `~/.config/` 路径并做错误回退 |
